# 部署发布 / Deployment & Publishing

## 发布前检查清单 / Pre-publish Checklist

- [ ] `package.json` 中 `version` 已递增
- [ ] `CHANGELOG.md` 已更新
- [ ] `publisher` 字段与 Marketplace Publisher ID 一致
- [ ] `pnpm run build` 编译无报错
- [ ] F5 调试测试核心功能正常
- [ ] `.vscodeignore` 配置正确，不包含多余文件

---

## 版本号规则 / Versioning Rules

遵循 [Semantic Versioning](https://semver.org/)：

```
主版本.次版本.修订号
MAJOR.MINOR.PATCH

MAJOR — 不兼容的 API 变更 / Breaking changes
MINOR — 新增功能（向后兼容）/ New features (backward compatible)
PATCH — Bug 修复 / Bug fixes

示例 / Examples:
  0.1.0 → 0.1.1  修复 Bug
  0.1.1 → 0.2.0  新增功能
  0.9.0 → 1.0.0  首个稳定版
```

## 打包流程 / Package Process

### 步骤 1：更新版本 / Update Version

```bash
# 修改 package.json 中的 version
# 修改 CHANGELOG.md 添加新版本记录
```

### 步骤 2：生产构建 / Production Build

```bash
pnpm run build
```

确认 `dist/extension.js` 大小合理（当前 ~73KB）。

### 步骤 3：打包 VSIX / Package VSIX

```bash
npx @vscode/vsce package --no-dependencies
```

输出示例：
```
DONE  Packaged: harmony-dev-tools-0.1.1.vsix (19 files, 97.08 KB)
```

### 步骤 4：本地验证 / Local Verification

```bash
# 在 VS Code 中安装本地 VSIX 验证
code --install-extension harmony-dev-tools-0.1.1.vsix
```

或在 VS Code 中：`Cmd+Shift+P` → `Extensions: Install from VSIX...`

## 发布到 Marketplace / Publish to Marketplace

### 方式一：网页上传（推荐）/ Web Upload (Recommended)

适合：不想配置 Azure DevOps 的开发者。

```
1. 打开 https://marketplace.visualstudio.com/manage
   使用微软账号登录

2. 首次发布需要创建 Publisher：
   ┌──────────────────────────────────────┐
   │  Create Publisher                     │
   │                                      │
   │  Name: HarmonyOS Dev Tools           │
   │  ID:   corn12138                     │
   │        ↑ 必须与 package.json 的       │
   │          publisher 字段一致           │
   │                                      │
   │  Description: (可选描述)              │
   │  Logo: resources/icons/icon.png      │
   └──────────────────────────────────────┘

3. 上传 VSIX：
   点击 + New extension → Visual Studio Code
   拖拽或选择 .vsix 文件上传

4. 等待验证：
   状态变化: Verifying → Published
   通常几分钟内完成

5. 更新版本：
   插件旁 ... → Update → 上传新 VSIX
```

### 方式二：命令行发布 / CLI Publish

适合：需要 CI/CD 自动化发布的场景。

**前提：创建 Azure DevOps 组织和 PAT**

```
1. 打开 https://aex.dev.azure.com
   登录微软账号，创建组织

2. 创建 Personal Access Token (PAT)：
   ┌──────────────────────────────────────┐
   │  Create New Token                     │
   │                                      │
   │  Name: vsce                          │
   │  Organization: All accessible orgs   │
   │  Expiration: 365 days                │
   │  Scopes:                             │
   │    Show all scopes                   │
   │    → Marketplace → ✓ Manage          │
   │                                      │
   │  [Create]                            │
   │                                      │
   │  ⚠️ 立刻复制 Token（只显示一次！）     │
   └──────────────────────────────────────┘

3. 命令行发布：
   # 登录
   npx @vscode/vsce login corn12138
   # 输入 PAT

   # 发布
   npx @vscode/vsce publish

   # 或一步完成
   npx @vscode/vsce publish --pat <YOUR_TOKEN>

   # 指定版本发布
   npx @vscode/vsce publish minor   # 自动递增次版本
   npx @vscode/vsce publish patch   # 自动递增修订号
```

## 推送到 GitHub / Push to GitHub

### 首次推送 / First Push

```bash
# 1. GitHub 上创建仓库（不添加 README/LICENSE/.gitignore）
#    https://github.com/new → harmony-dev-tools → Public → Create

# 2. 本地初始化并推送
git init
git branch -m main
git add -A
git commit -m "feat: initial release of HarmonyOS Dev Tools v0.1.0"
git remote add origin https://github.com/corn12138/harmony-dev-tools.git
git push -u origin main
```

### 日常推送 / Daily Push

```bash
git add <changed files>
git commit -m "feat/fix/docs: description"
git push origin main
```

### 创建 Release / Create Release

```bash
# 打标签
git tag v0.1.1
git push origin v0.1.1

# 通过 GitHub CLI 创建 Release（如已安装 gh）
gh release create v0.1.1 \
  --title "v0.1.1 — Bilingual Documentation" \
  --notes "Added Chinese/English bilingual README" \
  harmony-dev-tools-0.1.1.vsix
```

## 关键配置文件 / Key Config Files

### package.json — 核心字段

```jsonc
{
  "name": "harmony-dev-tools",          // npm 包名
  "displayName": "HarmonyOS Dev Tools", // 市场显示名
  "version": "0.1.1",                   // 版本号（每次发布必须递增）
  "publisher": "corn12138",             // 发布者 ID（必须与 Marketplace 一致）
  "engines": {
    "vscode": "^1.85.0"                // 最低 VS Code 版本
  },
  "activationEvents": [                 // 插件激活条件
    "onLanguage:arkts",
    "workspaceContains:**/build-profile.json5"
  ],
  "main": "./dist/extension.js",        // 入口文件（esbuild 输出）
  "contributes": {
    "commands": [...],                   // 21 个命令
    "languages": [...],                  // ArkTS 语言定义
    "grammars": [...],                   // TextMate 语法
    "snippets": [...],                   // 代码片段
    "jsonValidation": [...],            // JSON Schema
    "configuration": {...},              // 扩展设置
    "debuggers": [...],                 // 调试器
    "views": {...},                     // 设备面板
    "taskDefinitions": [...]            // hvigor 任务
  }
}
```

### .vscodeignore — VSIX 排除规则

```
.vscode/          # IDE 配置
src/              # 源码（只打包 dist/）
node_modules/     # 依赖（使用 --no-dependencies）
*.vsix            # 已有包
.gitignore
tsconfig.json
pnpm-lock.yaml
ARCHITECTURE.md
SUMMARY.md
docs/             # 文档目录
```

### .gitignore — Git 排除规则

```
node_modules/     # 依赖目录

.vscode-test/     # 测试输出
```
