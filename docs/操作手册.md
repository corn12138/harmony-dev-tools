## 一、安装与激活 / Installation & Activation

### 安装 / Install

**方式一：VS Code 市场（推荐）**
1. 打开 VS Code → 按 `Cmd+Shift+X` 打开扩展面板
2. 搜索 **"HarmonyOS Dev Tools"**
3. 点击 **Install**

**方式二：手动安装 VSIX**
1. 下载 `harmony-dev-tools-x.x.x.vsix` 文件
2. `Cmd+Shift+P` → `Extensions: Install from VSIX...`
3. 选择 `.vsix` 文件

### 激活条件 / Activation

插件在以下条件下自动激活：
- 工作区中存在 `build-profile.json5` 文件（标准鸿蒙项目）
- 打开 `.ets` 文件时

激活成功标志：状态栏左下角出现 **"HarmonyOS"** 字样。

### 2.1 代码补全 / Auto-completion

**触发方式**：在 `.ets` 文件中输入即可触发。

| 输入 / Type | 补全内容 / Completion |
|-------------|---------------------|
| `@` | 所有装饰器（V1 + V2 共 30 个） |

## 三、资源管理功能 / Resource Management

### 3.1 $r() 自动补全

在 `$r('` 后自动提示资源类型和键名：

```typescript
$r('app.         // → 提示 string, media, color, float, boolean...
$r('app.string.  // → 提示具体的字符串资源键名
```

### 3.2 Ctrl+Click 跳转

按住 `Ctrl`（Mac 为 `Cmd`）点击 `$r('app.string.title')`，直接跳转到资源定义文件。

### 3.3 资源校验

自动检测无效的 `$r()` 引用，在编辑器中显示警告波浪线。

## 四、构建与运行 / Build & Run

### 4.1 终端构建运行（推荐日常使用）

**操作**：`Cmd+Shift+P` → `HarmonyOS: Build & Run (Terminal)`

**执行流程**（在终端中可见）：
```
[1/4] Building HAP...         → chmod +x ./hvigorw && ./hvigorw assembleHap
[2/4] Locating HAP output...  → 查找 .hap 文件
[3/4] Installing to device... → hdc install -r xxx.hap
[4/4] Launching app...        → hdc shell aa start -a Ability -b Bundle
```

### 4.2 后台构建运行

**操作**：`Cmd+Shift+P` → `HarmonyOS: Build & Run on Device`

在后台执行，通过进度通知显示状态。完成后自动打开 UI Inspector。

### 4.3 其他构建命令

| 命令 / Command | 操作 / Action |
|---------------|-------------|
| `HarmonyOS: Build HAP` | 仅构建，不安装运行 |
| `HarmonyOS: Clean Build` | 清理构建缓存 |
| `HarmonyOS: Run on Device` | 安装已有 HAP 并运行 |
| `HarmonyOS: Stop App on Device` | 强制停止应用 |

## 五、调试功能 / Debugging

### 5.1 一键调试

**操作**：`Cmd+Shift+P` → `HarmonyOS: Debug App on Device`

**自动执行**：
1. 设置 HDC 端口转发（`hdc fport tcp:9230 tcp:9230`）
2. 以调试模式启动应用（`aa start -D`）
3. 通过 CDP 协议连接 VS Code 调试器

### 5.2 自定义 launch.json

在项目根目录 `.vscode/launch.json` 中添加：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "harmonyos",
      "request": "launch",
      "name": "Debug HarmonyOS App",
      "bundleName": "com.example.myapp",
      "abilityName": "EntryAbility",
      "debugPort": 9230
    }
  ]
}
```

| 参数 / Parameter | 说明 / Description | 默认值 / Default |
|-----------------|-------------------|-----------------|
| `bundleName` | 应用包名（为空自动检测）| 从 app.json5 读取 |
| `abilityName` | 入口 Ability 名 | `EntryAbility` |
| `deviceId` | 目标设备 ID（为空自动选择）| 首个在线设备 |
| `debugPort` | CDP 调试端口 | `9230` |

### 5.3 调试操作

连接后可使用 VS Code 标准调试功能：
- **设置断点**：点击行号左侧
- **单步执行**：F10（Step Over）/ F11（Step Into）
- **变量查看**：Variables 面板
- **调用堆栈**：Call Stack 面板
- **Debug Console**：输入表达式求值

## 八、模拟器管理 / Emulator Manager

> **v0.3.0 新功能** — 在 VS Code 中管理 DevEco Studio 模拟器。

### 8.1 启动模拟器

**操作**：`Cmd+Shift+P` → `HarmonyOS: Launch Emulator`

插件会自动扫描以下目录查找模拟器镜像：

| 平台 | 扫描路径 |
|------|---------|
| macOS | `~/Library/Huawei/DevEcoStudio/emulator/` |
| macOS | `~/Library/Huawei/Sdk/hms/emulator/` |
| Windows | `%LOCALAPPDATA%\Huawei\DevEcoStudio\emulator\` |
| Linux | `~/.Huawei/DevEcoStudio/emulator/` |

选择模拟器后，插件会：
1. 查找 emulator 可执行文件并启动
2. 轮询 HDC 等待模拟器上线（最长 60 秒）
3. 上线后提示 "Open Device Mirror" 一键进入镜像

### 8.2 停止模拟器

**操作**：`Cmd+Shift+P` → `HarmonyOS: Stop Emulator`

### 8.3 侧边栏集成

在 **HarmonyOS → Devices** 面板中，设备列表现在分为两组：

- **Devices** — 已连接的物理设备（点击打开 Device Mirror）
- **Emulators** — 检测到的模拟器镜像（绿色 = 运行中，灰色 = 已停止）

### 8.4 前置条件

- 模拟器镜像需先在 DevEco Studio 中创建（Tools > Device Manager > New Emulator）
- 部分模拟器可能需要 DevEco Studio 首次初始化才能被独立启动

## 十、实时诊断与 Quick Fix / Real-time Diagnostics (v0.4.0+)

> 解决 DevEco Studio 编译慢（20+ 分钟）的核心痛点 — 无需编译即可实时发现代码错误。

### 10.1 ArkTS 严格类型检查

编辑 `.ets` 文件时自动运行，以下问题会实时标红：

| 检查规则 | 严重级别 | Quick Fix |
|----------|----------|-----------|
| `: any` 类型使用 | ❌ Error | 替换为 `string` / `number` / `boolean` / `object` / `Record<string, Object>` |
| `: unknown` 类型使用 | ❌ Error | 替换为具体类型 |
| `as any` 强制转换 | ❌ Error | 移除 `as any` |
| 函数参数缺少类型标注 | ⚠️ Warning | 添加 `: string` 类型标注 |

### 10.2 状态管理陷阱检测

| 检查规则 | 严重级别 | Quick Fix |
|----------|----------|-----------|
| `@State` 修饰复杂对象（Array、Map、自定义类） | ⚠️ Warning | `@State` → `@Trace`（配合 `@ObservedV2`） |
| V1/V2 装饰器混用 | ❌ Error | 触发 V1→V2 完整迁移 |
| `@Link` 在 `@ComponentV2` 中使用 | ❌ Error | `@Link` → `@Param` |

### 10.3 性能反模式检测

| 检查规则 | 严重级别 | 说明 |
|----------|----------|------|
| `ForEach` 使用 | ℹ️ Info | 建议改用 `LazyForEach` 实现按需渲染 |
| `build()` 中调用 `fetch()` | ⚠️ Warning | 网络请求应移到生命周期方法 |
| `build()` 中调用 `setTimeout/setInterval` | ⚠️ Warning | 定时器应移到生命周期方法 |
| `build()` 中调用 `console.log` | ⚠️ Warning | 日志输出应移到生命周期方法 |
| `build()` 中调用 `JSON.parse/stringify` | ⚠️ Warning | 序列化操作应缓存到状态变量 |
| `build()` 中使用 `await` | ⚠️ Warning | 异步操作应移到生命周期方法 |

### 10.4 使用方式

- **自动启用**：打开 `.ets` 文件即生效，无需手动触发
- **关闭方式**：设置 `harmony.enableDiagnostics` 为 `false`
- **Quick Fix**：将光标放在红/黄波浪线上，按 `Cmd+.`（macOS）或 `Ctrl+.`（Windows/Linux）查看修复建议

## 十六、V1→V2 迁移工具 / V1 to V2 Migration

### 16.1 一键迁移

**操作**：`Cmd+Shift+P` → `HarmonyOS: Migrate V1 → V2 Decorators`

或在 `.ets` 文件中右键 → **Migrate V1 → V2 Decorators**

### 16.2 迁移对照表

| V1 装饰器 | V2 装饰器 | 自动迁移 |
|-----------|-----------|---------|
| `@Component` | `@ComponentV2` | 自动 |
| `@State` | `@Local` | 自动 |
| `@Prop` | `@Param` | 自动 |
| `@Provide` | `@Provider` | 自动 |
| `@Consume` | `@Consumer` | 自动 |
| `@Watch('handler')` | `@Monitor('prop')` | 自动（需手动调整参数）|
| `@Observed` | `@ObservedV2` | 自动 |
| `@ObjectLink` | `@Param` | 自动 + TODO 注释 |
| `@Link` | `@Param` + `@Event` | 需手动拆分 |

### 16.3 迁移后注意事项

1. **@Monitor**：V1 的 `@Watch` 监听变量名，V2 的 `@Monitor` 监听属性名，参数含义不同，请手动检查
2. **@ObjectLink**：迁移为 `@Param` 后需确认配合 `@ObservedV2` 使用
3. **@Link**：需手动拆分为 `@Param`（接收值）+ `@Event`（回调父组件）
4. **不要混用**：同一个文件中不要同时使用 V1 和 V2 装饰器

## 十七、API 兼容性检查 / API Compatibility Check

### 17.1 执行检查

**操作**：`Cmd+Shift+P` → `HarmonyOS: Check API Compatibility`

### 17.2 检查内容

| 检查项 / Check | 说明 / Description |
|---------------|-------------------|
| API 版本检测 | 从 build-profile.json5 读取 `targetSdkVersion` / `compileSdkVersion` / `compatibleSdkVersion` |
| API 12 特性检查 | V2 装饰器、CalendarPicker、AppStorageV2 等 |
| API 13 特性检查 | @Require、IsolatedComponent、EmbeddedComponent 等 |
| API 14 特性检查 | @Type、makeObserved、EffectComponent 等 |
| V1/V2 混用检测 | 同一文件中混用 V1 和 V2 → 警告 |
| 废弃 API 检测 | animateTo、@ohos.router、@ohos.fileio 等 15+ 废弃项 |
| hvigor-config.json5 | modelVersion 缺失或格式异常 → 警告 |
| 升级建议 | 当项目 API 低于当前已知公开版本时给出升级建议 |

### 17.3 结果查看

检查结果显示在 **Output** 面板 → **HarmonyOS Compatibility** 通道。

## 扩展设置 / Extension Settings

在 VS Code 设置中搜索 `harmony`：

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| `harmony.sdkPath` | `""` | SDK 路径（为空自动检测）|
| `harmony.hdcPath` | `""` | HDC 路径（为空自动检测）|
| `harmony.enableCodeLens` | `true` | 显示组件引用计数 |
| `harmony.enableResourceValidation` | `true` | $r() 引用校验 |
| `harmony.autoFormatOnSave` | `false` | 保存时自动格式化 |
| `harmony.devicePollInterval` | `5000` | 设备刷新间隔（毫秒）|
| `harmony.logLevel` | `"info"` | 日志级别 |
| `harmony.enableDiagnostics` | `true` | 实时 ArkTS 诊断（v0.4.0+）|
| `harmony.enablePerfLens` | `true` | 性能洞察 CodeLens（v0.4.0+）|
| `harmony.enableOhpmInsight` | `true` | OHPM 依赖洞察（v0.4.0+）|
