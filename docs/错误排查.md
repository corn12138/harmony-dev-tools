# 错误排查 / Troubleshooting

## 构建相关错误 / Build Errors

### 1. `zsh: command not found: hvigorw`

**现象 / Symptom**：
```
zsh: command not found: hvigorw
```

**原因 / Cause**：
`hvigorw` 是鸿蒙项目根目录下的本地脚本（类似 Android 的 `gradlew`），不是全局命令。不能像 `npm` 那样直接调用。

**解决 / Solution**：
使用相对路径 `./hvigorw` 替代 `hvigorw`。

**涉及文件 / Affected Files**：
- `src/build/runner.ts`
- `src/build/buildAndRun.ts`
- `src/build/terminalRunner.ts`

**修复代码 / Fix**：
```typescript
// 修复前 / Before
exec('hvigorw assembleHap --no-daemon');

// 修复后 / After
exec('./hvigorw assembleHap --no-daemon');
```

---

### 2. `SyntaxError: Invalid or unexpected token`

**现象 / Symptom**：
```
/path/to/hvigorw:1
#
SyntaxError: Invalid or unexpected token
```

**原因 / Cause**：
`hvigorw` 文件的第一行是 `#!/bin/bash`（shell shebang），它是一个 **shell 脚本**，不是 JavaScript 文件。用 `node ./hvigorw` 执行会导致 Node.js 尝试解析 shell 语法。

**解决 / Solution**：
去掉 `node` 前缀，直接执行 `./hvigorw`。

**修复代码 / Fix**：
```typescript
// 修复前 / Before
exec('node ./hvigorw assembleHap --no-daemon');

// 修复后 / After
exec('./hvigorw assembleHap --no-daemon');
```

---

### 3. `zsh: permission denied: ./hvigorw`

**现象 / Symptom**：
```
zsh: permission denied: ./hvigorw
```

**原因 / Cause**：
`hvigorw` 脚本没有可执行权限（`-rw-r--r--` 而非 `-rwxr-xr-x`）。可能是 Git 克隆时权限丢失，或项目生成时未设置权限。

**解决 / Solution**：
在执行构建前先添加可执行权限。

**修复代码 / Fix**：
```typescript
// 在构建命令前加 chmod
const commands = [
  'chmod +x ./hvigorw 2>/dev/null; ./hvigorw assembleHap --no-daemon',
  // ...
];
```

`2>/dev/null` 抑制"权限已存在"时的无关输出。

---

### 4. ArkTS 编译错误（项目代码）

**现象 / Symptom**：
```
ERROR: ArkTS:ERROR File: xxx.ets:42:10
 Type 'string' is not assignable to type 'number'.
 ... (85 errors)
```

**原因 / Cause**：
这是用户项目代码的编译错误，不是插件问题。

**判断方法 / How to Identify**：
- 错误信息中包含 `.ets` 文件路径 → 项目代码问题
- 错误信息中包含 `extension.ts` 或 `dist/` → 插件问题

**解决 / Solution**：
修复项目代码中的 ArkTS 类型错误。插件只负责调起构建流程，编译错误需要开发者自行修复。

## 设备相关错误 / Device Errors

### 5. `hdc: command not found`

**现象 / Symptom**：
```
hdc: command not found
```

**原因 / Cause**：
HDC 未安装或未加入系统 PATH。

**解决 / Solution**：

方式一：配置 HDC 路径
```json
// VS Code settings.json
{
  "harmony.hdcPath": "/path/to/hdc"
}
```

方式二：添加到 PATH
```bash
# macOS/Linux — 添加到 ~/.zshrc 或 ~/.bashrc
export PATH=$PATH:/path/to/harmonyos/sdk/toolchains

# 刷新
source ~/.zshrc
```

---

### 6. 设备未连接 / No Device Connected

**现象 / Symptom**：
```
[Empty] 0 targets
```
或设备面板显示空白。

**排查步骤 / Troubleshooting**：

```bash
# 1. 检查 HDC 是否能识别设备
hdc list targets

# 2. 如果为空，检查：
#    - USB 线是否连接
#    - 设备是否开启"开发者模式"
#    - 设备是否信任当前电脑

# 3. 重启 HDC 服务
hdc kill
hdc start

# 4. 重启设备 USB 调试
#    设置 → 开发者选项 → 关闭/开启 USB 调试
```

---

### 7. HAP 安装失败 / HAP Install Failed

**现象 / Symptom**：
```
install error: signature verification failed
```

**原因 / Cause**：
设备的签名验证与 HAP 包的签名不匹配。

**解决 / Solution**：
```bash
# 使用 -r 参数覆盖安装
hdc install -r path/to/app.hap

# 如果仍然失败，先卸载再安装
hdc shell bm uninstall -n com.example.app
hdc install path/to/app.hap
```

## 调试相关错误 / Debug Errors

### 8. 端口转发失败 / Port Forward Failed

**现象 / Symptom**：
```
fport failed: port already in use
```

**原因 / Cause**：
之前的调试会话没有正确清理端口转发。

**解决 / Solution**：
```bash
# 查看现有端口转发
hdc fport ls

# 删除指定端口转发
hdc fport rm tcp:9230 tcp:9230

# 重新建立
hdc fport tcp:9230 tcp:9230
```

---

### 9. 调试器无法连接 / Debugger Cannot Attach

**现象 / Symptom**：
```
Cannot connect to runtime process, timeout after 10000ms
```

**原因 / Cause**：
应用未在调试模式下启动，或 CDP 端口不正确。

**排查步骤 / Troubleshooting**：

```bash
# 1. 确认应用以调试模式启动
hdc shell "aa start -D -a EntryAbility -b com.example.app"
# -D 参数表示调试模式

# 2. 确认端口转发正常
hdc fport ls

# 3. 确认本地端口可用
curl http://localhost:9230/json
# 应返回 JSON 调试信息

# 4. 检查 launch.json 配置
{
  "type": "harmonyos",
  "request": "launch",
  "debugPort": 9230  // 与 fport 端口一致
}
```

## 插件相关错误 / Extension Errors

### 10. 插件未激活 / Extension Not Activated

**现象 / Symptom**：
状态栏没有显示 "HarmonyOS"，命令面板搜索不到 "HarmonyOS" 命令。

**排查步骤 / Troubleshooting**：

```
1. 检查是否打开了鸿蒙项目
   → 工作区必须包含 build-profile.json5 文件

2. 检查是否打开了 .ets 文件
   → onLanguage:arkts 也会触发激活

3. 检查扩展是否已安装
   → Cmd+Shift+X → 搜索 "HarmonyOS"

4. 查看输出日志
   → View → Output → 选择 "HarmonyOS"
```

---

### 11. 命令执行报错 / Command Execution Error

**现象 / Symptom**：
```
Cannot find module './build/terminalRunner'
```

**原因 / Cause**：
esbuild 打包时可能遗漏了模块，或 `dist/extension.js` 未更新。

**解决 / Solution**：
```bash
# 重新构建
pnpm run build

# 如果使用本地 VSIX，重新打包安装
npx @vscode/vsce package --no-dependencies
code --install-extension harmony-dev-tools-x.x.x.vsix
```

## 发布相关错误 / Publishing Errors

### 12. Azure DevOps 无法访问 / Cannot Access Azure DevOps

**现象 / Symptom**：
打开 `dev.azure.com` 重定向到 `portal.azure.com`。

**解决 / Solution**：
直接访问 `https://aex.dev.azure.com`，这个地址会跳到组织管理页面。

如果仍然有问题，使用**网页上传方式**发布：
1. 打开 https://marketplace.visualstudio.com/manage
2. 点 + New extension → Visual Studio Code
3. 上传 .vsix 文件

---

### 13. VSIX 发布验证失败 / VSIX Verification Failed

**可能原因 / Possible Causes**：
- `publisher` 字段与 Marketplace Publisher ID 不一致
- `package.json` 缺少必要字段（`name`, `version`, `engines`）
- 图标文件缺失（`icon` 字段指向不存在的文件）

**解决 / Solution**：
```bash
# 检查 VSIX 内容
npx @vscode/vsce ls

# 确认 publisher 一致
grep '"publisher"' package.json
# 应输出: "publisher": "corn12138"
```

## 快速诊断流程 / Quick Diagnosis Flow

```
问题出现
    │
    ├── 构建失败？
    │   ├── command not found → 检查 ./hvigorw 路径
    │   ├── permission denied → chmod +x ./hvigorw
    │   ├── SyntaxError       → 不要用 node 执行 hvigorw
    │   └── ArkTS errors     → 项目代码问题，非插件问题
    │
    ├── 设备问题？
    │   ├── 设备未识别 → hdc list targets / 检查 USB
    │   ├── 安装失败   → hdc install -r / 签名问题
    │   └── hdc 未找到 → 配置 harmony.hdcPath
    │
    ├── 调试问题？
    │   ├── 端口占用   → hdc fport rm + 重新 fport
    │   ├── 连接超时   → 确认 -D 参数 + 端口转发
    │   └── 断点不命中 → 检查 sourcemap
    │
    └── 插件问题？
        ├── 未激活     → 检查 build-profile.json5 是否存在
        ├── 命令报错   → pnpm run build 重新编译
        └── 发布失败   → 检查 publisher 一致性
```
