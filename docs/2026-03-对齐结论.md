# 2026-03 鸿蒙社区与官方文档对齐结论

## 项目现状

当前插件主能力已经覆盖：

- ArkTS 语言支持、V1/V2 迁移、实时诊断、资源索引
- 构建运行、设备调试、日志、镜像、模拟器
- 工程文件视图、依赖洞察、配置文件 hover、文档搜索

主要老化点集中在：

- `build-profile.json5` 仍偏向旧版 `compileSdkVersion`
- `oh-package.json5` 缺少新版常见 `modelVersion`
- API 检测只支持纯数字版本
- 环境检查没有覆盖官方 Command Line Tools
- 文档入口缺少“版本说明 / 知识地图 / 命令行工具”快捷跳转

## 官方资料信号

- 官方首页已进入 [HarmonyOS 6](https://developer.huawei.com/consumer/cn/) 时代。
- 官方知识体系已提供 [HarmonyOS 知识地图](https://developer.huawei.com/consumer/cn/app/knowledge-map/)。
- 官方文档提供独立的 [Command Line Tools](https://developer.huawei.com/consumer/en/doc/harmonyos-guides/ide-commandline-get) 页面，明确包含 `sdkmgr`、`codelinter`、`ohpm`。
- 官方版本说明页为 [Release Notes](https://developer.huawei.com/consumer/en/doc/harmonyos-releases/overview-allversion)。
- 2026 年 1 月开发者月刊提到 [HarmonyOS 6.0.2(22)](https://developer.huawei.com/consumer/cn/app/briefing/20260131) 已发布，并强调 DevEco Studio / Code Linter 能力升级。

## 社区信号

- 官方样例仓库已经广泛使用 `targetSdkVersion` 与 `buildModeSet`，例如 [ImageShow](https://gitee.com/harmonyos_samples/ImageShow) 等样例。
- 官方样例仓库中的 `oh-package.json5` 已常见 `modelVersion: "5.0.5"`，例如 [Hello World](https://gitee.com/harmonyos_samples/hello-world)。
- 社区/样例 issue 反复出现版本对不上的问题，例如 [hvigorVersion 现在对不上了](https://gitee.com/harmonyos/samples/issues/ICUVMU)。

## 已落地调整

- API/版本解析同时支持 `targetSdkVersion`、`compileSdkVersion`、`compatibleSdkVersion`
- 支持 `6.0.0(20)`、`6.0.2(22)` 这类字符串版本提取 API 级别
- 更新 `build-profile.schema.json`，补充 `targetSdkVersion`、`buildModeSet`
- 更新 `oh-package.schema.json` 与 hover，补充 `modelVersion`
- 新建项目模板改为输出新版构建配置
- 环境检查新增 Command Line Tools 检测
- 文档搜索新增官方快捷入口：版本说明、知识地图、Command Line Tools、Downloads
- README / 操作手册 / ROADMAP 同步更新

## 后续建议

- 继续补充 API 15+ / 20+ 的组件与装饰器元数据，而不是只停留在工程配置识别层
- 增加 `targetSdkVersion` 对应的版本文档推荐逻辑
- 把社区高频 issue 中的预览器差异、hvigor 版本错配整理成 FAQ 或诊断规则
