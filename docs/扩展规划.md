## 短期规划 / Short-term Roadmap (v0.2.x - v0.3.x)

### 1. 智能诊断 / Intelligent Diagnostics

**目标 / Goal**：在编辑器中实时显示 ArkTS 语法和类型错误。

```
现状：用户需要执行构建才能看到编译错误
目标：在保存文件时自动执行轻量级语法检查

实现思路 / Approach:
├── 利用 hvigorw 的增量编译能力
├── 解析 hvigor 输出的错误信息
├── 映射到 VS Code DiagnosticCollection
└── 在编辑器中显示红色波浪线 + 错误提示
```

### 2. 热重载 / Hot Reload

**目标 / Goal**：保存 .ets 文件后自动推送变更到设备。

```
实现思路 / Approach:
├── 监听 .ets 文件保存事件
├── 增量编译变更的模块
├── 通过 HDC 推送到设备
└── 触发应用内热重载
```

### 3. 多设备支持 / Multi-device Support

**目标 / Goal**：同时连接多个设备，支持选择目标设备。

```
实现思路 / Approach:
├── DeviceTreeView 支持多设备展示
├── 每个设备节点显示设备名、类型、状态
├── 右键菜单：安装 / 运行 / 调试 / 截图
└── 构建运行时弹出设备选择器
```

### 4. 性能分析面板 / Performance Profiler

**目标 / Goal**：可视化 HarmonyOS 应用的性能数据。

```
实现思路 / Approach:
├── 通过 HDC 获取 CPU/内存/帧率数据
├── WebView 面板实时绘制图表
├── 标记性能瓶颈（掉帧、内存泄漏）
└── 导出性能报告
```

## 中期规划 / Mid-term Roadmap (v0.4.x - v0.6.x)

### 5. ArkTS Language Server

**目标 / Goal**：实现完整的 ArkTS 语言服务器，提供类似 TypeScript 的开发体验。

```
功能 / Features:
├── 实时类型检查（不依赖构建）
├── 重构支持（重命名、提取方法）
├── 查找引用、查找定义
├── 自动修复（Quick Fix）
└── 工作区符号搜索

技术方案 / Technical Approach:
├── 基于 LSP（Language Server Protocol）
├── 可考虑复用 TypeScript compiler API
├── 添加 ArkTS 特有的装饰器和组件语义
└── 独立进程运行，不阻塞编辑器
```

### 6. 组件可视化编辑器 / Visual Component Editor

**目标 / Goal**：拖拽式 ArkUI 布局编辑器。

```
功能 / Features:
├── 组件面板：拖拽 Column/Row/Text/Button 等
├── 属性编辑器：可视化修改样式
├── 双向同步：编辑器 ↔ 代码 实时同步
└── 预览：实时渲染 ArkUI 布局

技术方案 / Technical Approach:
├── WebView 面板 + Canvas 渲染
├── AST 解析 .ets 文件
├── 修改 AST 后回写代码
└── FileSystemWatcher 监听代码变更反向更新
```

### 7. 测试框架集成 / Test Framework Integration

**目标 / Goal**：集成 HarmonyOS 测试框架。

```
功能 / Features:
├── Test Explorer 面板显示测试用例
├── 一键运行/调试单个测试
├── 测试覆盖率显示
└── 测试结果在编辑器内联显示

技术方案 / Technical Approach:
├── 解析 @ohos/hypium 测试配置
├── 通过 hdc 在设备上运行测试
├── 解析 JUnit XML 格式结果
└── 映射到 VS Code Test API
```

## 扩展点使用指南 / Extension Point Guide

### 当前已定义的 8 个扩展点 / 8 Extension Points

第三方 VS Code 插件可以通过 Public API 扩展以下功能：

#### 1. Snippet — 代码片段扩展

```typescript
const api = vscode.extensions.getExtension('corn12138.harmony-dev-tools')?.exports;

api.registerSnippetContributor({
  id: 'my-snippets',
  language: 'arkts',
  snippets: [
    {
      prefix: 'mycomp',
      body: ['@Component', 'struct ${1:MyComponent} {', '  build() {', '    $0', '  }', '}'],
      description: 'My custom component template'
    }
  ]
});
```

#### 2. Schema — JSON Schema 扩展

```typescript
api.registerSchemaContributor({
  id: 'my-schema',
  filePattern: '**/my-config.json5',
  schema: {
    type: 'object',
    properties: {
      name: { type: 'string', description: 'Config name' }
    }
  }
});
```

#### 3. Template — 项目模板扩展

```typescript
api.registerTemplateContributor({
  id: 'iot-template',
  name: 'IoT Device App',
  description: 'Template for HarmonyOS IoT applications',
  category: 'app',
  async generate(targetDir, options) {
    // 生成项目文件...
  }
});
```

#### 4. LintRule — Lint 规则扩展

```typescript
api.registerLintRule({
  id: 'no-hardcoded-string',
  name: 'No Hardcoded Strings',
  severity: vscode.DiagnosticSeverity.Warning,
  check(document) {
    // 检查并返回 Diagnostic[]
    return [];
  }
});
```

#### 5. DeviceAdapter — 设备适配器扩展

```typescript
api.registerDeviceAdapter({
  id: 'wifi-device',
  name: 'WiFi Device Adapter',
  protocol: 'wifi',
  async discover() { /* 发现设备 */ return []; },
  async install(device, hapPath) { /* 安装 */ },
  async uninstall(device, bundleName) { /* 卸载 */ },
  async shell(device, command) { /* 执行命令 */ return ''; }
});
```

#### 6. BuildTask — 构建任务扩展

```typescript
api.registerBuildTask({
  id: 'custom-build',
  taskType: 'custom',
  label: 'Custom Build',
  async execute(workspace, options) {
    return { success: true, duration: 1000, outputPath: '/path/to/output.hap' };
  }
});
```

#### 7. PreviewRenderer — 预览渲染器扩展

```typescript
api.registerPreviewRenderer({
  id: 'canvas-renderer',
  name: 'Canvas Renderer',
  supportedComponents: ['Canvas', 'XComponent'],
  async render(source, ctx) {
    return '<div>Rendered preview HTML</div>';
  }
});
```

#### 8. CodeAction — 代码操作扩展

```typescript
api.registerCodeAction({
  id: 'wrap-with-if',
  title: 'Wrap with if condition',
  kind: vscode.CodeActionKind.Refactor,
  applicableWhen(document, range) { return !range.isEmpty; },
  async execute(document, range) {
    const edit = new vscode.WorkspaceEdit();
    // 编辑逻辑...
    return edit;
  }
});
```

## 事件监听 / Event Subscription

第三方插件可以订阅事件：

```typescript
const api = vscode.extensions.getExtension('corn12138.harmony-dev-tools')?.exports;

// 监听构建事件
api.onBuildStarted(({ task, module }) => {
  console.log(`Build started: ${task}`);
});

api.onBuildCompleted(({ task, success, duration }) => {
  console.log(`Build ${success ? 'succeeded' : 'failed'} in ${duration}ms`);
});

// 监听设备事件
api.onDeviceChanged(({ id, name, type, status, change }) => {
  console.log(`Device ${id}: ${change} (${status ?? 'unknown'})`, name, type);
});

// 监听项目检测
api.onProjectDetected(({ rootPath, modules }) => {
  console.log(`HarmonyOS project detected at ${rootPath}`);
});
```

## 贡献指南 / Contributing

### 如何贡献 / How to Contribute

1. **Fork** 仓库：https://github.com/corn12138/harmony-dev-tools
2. 创建功能分支：`git checkout -b feat/my-feature`
3. 编写代码并测试
4. 提交 PR：描述你的改动

### 贡献方向 / Contribution Areas

| 方向 / Area | 难度 / Difficulty | 说明 / Description |
|------------|-----------------|-------------------|
| 代码片段 | 低 / Easy | 添加更多 ArkUI 组件片段 |
| 文档翻译 | 低 / Easy | 改进中英文文档 |
| Bug 修复 | 中 / Medium | 修复已知问题 |
| 新 Provider | 中 / Medium | 添加新的语言特性支持 |
| 测试集成 | 高 / Hard | 集成 HarmonyOS 测试框架 |
| Language Server | 高 / Hard | 实现 ArkTS LSP |
