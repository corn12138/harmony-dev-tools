# 开发指南 / Development Guide

## 环境准备 / Environment Setup

### 必需工具 / Required Tools

| 工具 / Tool | 版本 / Version | 用途 / Purpose |

```bash
pnpm install

# 3. 编译
pnpm run build
```

## 开发工作流 / Development Workflow

### 日常开发 / Daily Development

```bash
# 启动监听模式（文件变化自动编译）
pnpm run watch
```

然后在 VS Code 中：
1. 按 **F5** 启动 Extension Development Host
2. 新窗口中打开一个鸿蒙项目测试功能
3. 修改代码后，在新窗口中按 `Cmd+Shift+P` → `Developer: Reload Window` 重载

### 调试插件 / Debugging the Extension

`.vscode/launch.json` 已配置好：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Run Extension",
      "type": "extensionHost",
      "request": "launch",
      "args": ["--extensionDevelopmentPath=${workspaceFolder}"],
      "outFiles": ["${workspaceFolder}/dist/**/*.js"],
      "preLaunchTask": "npm: watch"
    }
  ]
}
```

- 在 `src/` 下的 TypeScript 文件中设置断点
- F5 启动后断点会生效
- 在 Debug Console 中可以查看 `Logger` 输出

## 添加新功能 / Adding New Features

### 添加新命令 / Add a New Command

**第一步**：在 `src/utils/constants.ts` 中添加命令 ID

```typescript
export const COMMANDS = {
  // ... 已有命令
  MY_NEW_COMMAND: 'harmony.myNewCommand',
} as const;
```

**第二步**：在 `package.json` 的 `contributes.commands` 中注册

```json
{
  "command": "harmony.myNewCommand",
  "title": "HarmonyOS: My New Command",
  "category": "HarmonyOS"
}
```

**第三步**：在 `src/extension.ts` 的 `registerLazyCommands()` 中注册处理函数

```typescript
lazyCommand(COMMANDS.MY_NEW_COMMAND, async () => {
  const { myNewFeature } = await import('./myModule/myFeature');
  await myNewFeature();
});
```

**第四步**：实现功能模块

```typescript
// src/myModule/myFeature.ts
import * as vscode from 'vscode';

export async function myNewFeature(): Promise<void> {
  vscode.window.showInformationMessage('My new feature works!');
}
```

### 添加新的语言 Provider / Add a Language Provider

在 `registerLazyLanguageFeatures()` 中添加：

```typescript
context.subscriptions.push(
  vscode.languages.registerXxxProvider(LANGUAGE_ID, {
    async provideXxx(document, position) {
      const { provideXxx } = await import('./language/myProvider');
      return provideXxx(document, position);
    },
  })
);
```

### 添加新的扩展点 / Add an Extension Point

1. 在 `src/core/registry.ts` 的 `ExtensionPoints` 中添加新的扩展点名称
2. 在 `src/core/api.ts` 中定义接口并添加注册方法
3. 在对应模块中使用 `registry.getAll(point)` 获取注册的扩展

## 构建和打包 / Build & Package

### 开发构建 / Development Build

```bash
# 监听模式
pnpm run watch

# 单次构建
pnpm run build
```

### 生产打包 / Production Package

```bash
# 1. 生产构建（esbuild minify）
pnpm run build

# 2. 打包 VSIX
npx @vscode/vsce package --no-dependencies

# 输出: harmony-dev-tools-x.x.x.vsix
```

### esbuild 配置说明 / esbuild Config

```javascript
// package.json scripts
{
  "build": "esbuild src/extension.ts --bundle --platform=node --target=node18 --outfile=dist/extension.js --external:vscode --minify --sourcemap",
  "watch": "esbuild src/extension.ts --bundle --platform=node --target=node18 --outfile=dist/extension.js --external:vscode --sourcemap --watch"
}
```

关键配置：
- `--bundle`：打包所有依赖到单个文件
- `--platform=node`：Node.js 平台
- `--external:vscode`：不打包 vscode 模块（由 VS Code 提供）
- `--minify`：生产模式压缩
- `--sourcemap`：源码映射（调试用）
- `--no-dependencies`：VSIX 不包含 node_modules

## 代码规范 / Code Conventions

### 命名规范 / Naming

| 类型 / Type | 规范 / Convention | 示例 / Example |
|------------|------------------|---------------|
| 文件名 | camelCase | `terminalRunner.ts` |
| 类名 | PascalCase | `HarmonyDebugAdapter` |
| 函数名 | camelCase | `buildAndRun()` |
| 常量 | UPPER_SNAKE_CASE | `COMMANDS.BUILD_HAP` |
| 事件名 | domain:action | `'build:completed'` |
| 扩展点 | harmony.category | `'harmony.templates'` |

### 模块规范 / Module Conventions

1. **每个模块独立** — 不直接 import 其他功能模块
2. **通过 EventBus 通信** — 模块间用事件解耦
3. **懒加载优先** — 使用 `await import()` 延迟加载
4. **类型安全** — 所有公开接口必须有 TypeScript 类型
5. **资源清理** — 所有 Disposable 必须注册到 `context.subscriptions`

### Git 提交规范 / Commit Convention

```
<type>: <description>

类型 / Types:
  feat:     新功能
  fix:      修复 Bug
  docs:     文档更新
  chore:    构建/配置变更
  refactor: 重构
  style:    代码格式
  test:     测试

示例 / Examples:
  feat: add terminal-based build & run
  fix: change hvigorw to ./hvigorw for shell execution
  docs: bilingual README (English + Chinese)
```

## 常见开发问题 / Common Dev Issues

### Q: 修改代码后不生效？

确认正在运行 `pnpm run watch`，然后在 Extension Development Host 窗口中 `Cmd+Shift+P` → `Developer: Reload Window`。

### Q: 断点不命中？

确认 `launch.json` 中 `outFiles` 指向 `dist/**/*.js`，且 esbuild 生成了 sourcemap。

### Q: 新命令不显示？

1. 检查 `package.json` 的 `contributes.commands` 是否注册
2. 检查 `activationEvents` 是否包含对应的触发条件
3. 重新加载窗口

### Q: VSIX 打包文件太大？

检查 `.vscodeignore`，确认 `src/`、`node_modules/`、`*.vsix` 等已排除。
