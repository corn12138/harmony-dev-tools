## 微内核详解 / Microkernel Detail

### EventBus — 事件总线

类型安全的发布-订阅系统，所有模块间通信的唯一通道。

```typescript
// 事件类型定义（编译时类型检查）
interface HarmonyEvents {
  'project:detected':     { rootPath: string; modules: string[] };
  'project:configChanged': { file: string; type: string };
  'build:started':        { task: string; module?: string };
  'build:completed':      { task: string; success: boolean; duration: number };
  'build:error':          { message: string; file?: string; line?: number };
  'device:connected':     { id: string; name: string; type: string };
  'device:disconnected':  { id: string };
  'device:appInstalled':  { deviceId: string; bundleName: string };
  'resource:changed':     { type: string; path: string };
  'resource:indexRebuilt': { count: number };
  'language:diagnostics': { file: string; errors: number; warnings: number };
  'extension:activated':  { id: string };
  'extension:registered': { point: string; contributor: string };
}
```

**核心方法 / Core Methods**：

| 方法 / Method | 作用 / Purpose |
|---------------|---------------|
| `on(event, listener)` | 订阅特定事件 / Subscribe to event |
| `emit(event, data)` | 发布事件 / Publish event |
| `onPattern(pattern, listener)` | 通配符订阅 / Wildcard subscribe |
| `dispose()` | 清理所有监听器 / Cleanup |

**数据流示例 / Data Flow Example**：

```
用户点击 "Build & Run"
    │
    ▼
terminalRunner.ts
    │ emit('build:started', { task: 'assembleHap' })
    │
    ▼
./hvigorw assembleHap --no-daemon
    │
    ├── 成功 → emit('build:completed', { success: true, duration: 15000 })
    │           │
    │           ▼
    │       hdc install → emit('device:appInstalled', { bundleName, deviceId })
    │           │
    │           ▼
    │       hdc shell aa start → 应用启动
    │
    └── 失败 → emit('build:error', { message: '编译失败' })
```

### Registry — 服务注册表

扩展点管理系统，支持动态注册和查询。

```typescript
// 8 个扩展点
ExtensionPoints = {
  SNIPPET,      // 代码片段扩展
  SCHEMA,       // JSON Schema 扩展
  TEMPLATE,     // 项目模板扩展
  LINT_RULE,    // Lint 规则扩展
  DEVICE,       // 设备适配器扩展
  BUILD_TASK,   // 构建任务扩展
  PREVIEW,      // 预览渲染器扩展
  CODE_ACTION,  // 代码操作扩展
}
```

**注册流程 / Registration Flow**：

```
第三方插件                   Registry                    EventBus
    │                          │                           │
    │ register(TEMPLATE, {     │                           │
    │   id: 'iot-template',   │                           │
    │   name: 'IoT App',     │                           │
    │   generate: ...         │                           │
    │ })                      │                           │
    │ ──────────────────────► │                           │
    │                          │ emit('extension:registered', │
    │                          │   { point: TEMPLATE,      │
    │                          │     contributor: 'iot' }) │
    │                          │ ─────────────────────────►│
    │                          │                           │
    │ ◄─── Disposable ─────── │                           │
    │   (用于取消注册)          │                           │
```

### ModuleManager — 模块管理器

管理模块的生命周期：注册 → 激活 → 停用。

```typescript
interface HarmonyModule {
  readonly id: string;                    // 模块唯一标识
  readonly dependencies?: string[];        // 依赖的其他模块
  activate(context: ModuleContext): Promise<void>;  // 激活
  deactivate(): Promise<void>;            // 停用
  readonly isActive: boolean;
}
```

**依赖解析 / Dependency Resolution**：

```
激活模块 A
    │
    ├── A 依赖 B？
    │   ├── B 已激活 → 跳过
    │   └── B 未激活 → 先激活 B
    │       ├── B 依赖 C？
    │       │   └── ... (递归)
    │       └── B.activate()
    │
    └── A.activate()
```

## 文件依赖关系 / File Dependencies

```
extension.ts (入口)
├── core/eventBus.ts        ← 零依赖
├── core/registry.ts        ← 依赖 eventBus
├── core/module.ts          ← 依赖 eventBus, registry, logger
├── core/api.ts             ← 依赖 registry, eventBus
├── utils/constants.ts      ← 零依赖
├── utils/config.ts         ← 依赖 vscode
├── utils/logger.ts         ← 依赖 vscode
│
├── language/*              ← 依赖 utils/constants
├── resource/*              ← 依赖 utils/constants, utils/config
├── build/*                 ← 依赖 utils/config, utils/constants
├── debug/*                 ← 依赖 utils/config, utils/constants
├── device/*                ← 依赖 utils/config, utils/constants
└── project/*               ← 依赖 utils/constants
```

**关键原则 / Key Principle**：所有功能模块只依赖 `utils/` 和 `core/`，不互相直接依赖。模块间通信只通过 EventBus。
